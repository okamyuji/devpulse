Read the implementation plan at docs/implementation-plan.md and execute the tasks sequentially using TDD. For each task:

1. Write the failing test first
2. Verify it fails
3. Write the minimal implementation to pass
4. Verify all tests pass
5. Commit with a descriptive message

Track progress by checking off completed steps in the plan. Before starting, read CONCEPT.md for product context and CLAUDE.md for project conventions.

Current task status: check the plan for unchecked (`- [ ]`) items to find where to resume.
